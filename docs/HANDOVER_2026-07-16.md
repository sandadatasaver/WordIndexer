## Current release

**v0.5.0 — Hierarchical Entries and Cross-References**

The stable `main` branch contains the tested core, automatic INDEX fields, analysis reports, optional table-cell indexing, hierarchical entries, `See`, and `See also` references. The `develop` branch is for active Version 2 work.

## Validated capabilities

WordIndexer can now:

- Load a Microsoft Word `.docx` manuscript.
- Load JSON dictionaries with aliases, hierarchy, and cross-reference metadata.
- Search every occurrence of enabled terms.
- Write canonical `index_as` terms.
- Resolve overlapping matches globally.
- Handle terms split across formatted runs.
- Detect an explicit TOC boundary or use a first-chapter fallback.
- Traverse body paragraphs and top-level tables in document order with `--include-tables`.
- Insert Word-compatible XE fields after matched text.
- Preserve visible text and surrounding run formatting.
- Append an `Index` heading and Word `INDEX` field automatically.
- Generate hierarchical XE fields.
- Generate `See` and `See also` fields.
- Save a new indexed DOCX without overwriting the source.
- Produce read-only console and JSON analysis reports.

## Current scope limitations

The engine does not yet search:

- Headers and footers.
- Footnotes and endnotes.
- Text boxes and other embedded Word stories.

These require separate traversal and regression tests.

## How to run

```powershell
python -m pytest -q
python book_indexer.py --version
python book_indexer.py inspect input/sample.docx
python book_indexer.py analyze input/sample.docx dictionaries/technology/powershell.json
python book_indexer.py index input/sample.docx dictionaries/technology/powershell.json output/sample_indexed.docx
```

Useful switches:

- `--include-tables` — include table-cell paragraphs.
- `--no-index-field` — omit the automatic visible INDEX field.
