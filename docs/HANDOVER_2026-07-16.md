## Current release

**v0.3.0 — Analysis and Coverage Reports**

The stable `main` branch contains the tested core and automatic INDEX-field feature. The `develop` branch is for active Version 2 work.

## Validated capabilities

WordIndexer can now:

- Load a Microsoft Word `.docx` manuscript.
- Load a JSON dictionary.
- Search every occurrence of enabled terms.
- Search aliases and write canonical `index_as` terms.
- Resolve overlapping matches globally.
- Handle terms split across formatted runs.
- Detect an explicit TOC boundary when exposed by Word.
- Fall back to the first `Heading 1` beginning with `Chapter 1`.
- Insert Word-compatible XE fields after matched text.
- Preserve visible text and surrounding run formatting.
- Append an `Index` heading and Word `INDEX` field automatically.
- Save a new indexed DOCX without overwriting the source.
- Produce read-only console analysis reports.
- Export analysis reports as JSON.

## Current Version 2 work

Dry-run analysis and coverage reporting are implemented on `develop`.

Example:

```powershell
python book_indexer.py analyze `
    input/sample.docx `
    dictionaries/technology/powershell.json `
    --json-output output/sample_analysis.json
```

The report includes:

- Body boundary.
- Ignored paragraphs.
- TOC detection method.
- Dictionary coverage.
- Missing terms.
- Total occurrences.
- Overlaps resolved.
- Per-term counts.

## Current scope limitations

The engine currently searches normal body paragraphs and headings. It does not yet search:

- Table cells.
- Headers and footers.
- Footnotes and endnotes.
- Text boxes and other embedded Word stories.

The project owner selected release-first scope before expanding these areas.

## How to run

```powershell
python -m pytest -q
python book_indexer.py --version
python book_indexer.py inspect input/sample.docx
python book_indexer.py analyze input/sample.docx dictionaries/technology/powershell.json
python book_indexer.py index input/sample.docx dictionaries/technology/powershell.json output/sample_indexed.docx
```

The optional `--no-index-field` switch disables automatic INDEX-field insertion.
